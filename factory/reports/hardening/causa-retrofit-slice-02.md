that opt in only on the `g2-editorial-counter` profile (Causa's exclusive

verifiable axis.

### Why R2 + R3 together rather than separate slices

The two retrofits are wholly independent at the surface level (cases-magazine
vs narrative-band) but share three pieces of infrastructure that must extend
in lock-step:
- `MOTION_PROFILES` flag set (one entry per pattern)
- views.py context surface (one ctx key per pattern)
- `_base.html` body data-attribute emitter (one `{% if %}` per pattern)

Splitting them would require landing the infrastructure twice, with two
walker passes and two scorecard regenerations. A single slice 02 commit is
the cleaner shape — and it matches the priority-plan §1.5 implementation
order (R2 + R3 named as the same A.6b walk pass).

### Why this slice does NOT touch Cornice

Cornice is `published_live` since 2026-05-01. Its frozen-sibling regression
is a hard constraint — any byte-level change to Cornice's render forces a
public-flip re-walk (5 locales × 5 routes = 25 walks) plus a 4-sibling
cluster regression (Pragma · Fiscus · Solaria · Continua). Slice 02 ships
Causa-only diversification; Cornice stays at the LF-2 1st-occupant baseline
(0 patterns enabled · static throughout).

### Why this slice does NOT widen multilingual

Causa is `tier=draft` and IT-only by design. Workflow C (1 → 5 locales) is a
separate brief gated on user handshake. The slice 02 changes do introduce
new IT copy (the 4 citation snippets · the 6 chronotick year-labels · the
toggle label "Vedi la massima n. X") — these will propagate to EN/FR/ES/AR
when workflow C runs, with the same locale-parity contract that AC-V3
established for the CTA verb-class (verbatim translation; no locale
collapses onto Cornice's strings).

## §2 · Implementation

### §2.1 · R2 · EVID-3 case-citation-pop

**What changed**: each of the 4 LF-2 magazine-grid case cards on Causa's
home gains a native `<details class="citation-pop" data-evid3>` element
containing a forensic-publication-register massima citation. The element
sits OUTSIDE the card-link `<a>` (so the citation toggle is not nested
inside an interactive ancestor) and INSIDE the `<article class="card">`
(so it inherits the card's grid placement and border treatment).

The `<summary>` is styled as a button with a label (`Vedi la massima n. X`)
and a `+` indicator that rotates 45° on `[open]`. The native `::-webkit-
details-marker` is hidden; the summary keeps its keyboard semantics (Enter/
Space toggles · `:focus-visible` outline in cluster gold).

The expanded body is GT Sectra italic 16px on cream — the same editorial-
serif italic register that pull-quotes use in the LF-2 narrative — so the
citation reads as "from the studio's massimario interno" rather than as
generic UI copy. The em-tag inside each citation lands on the principle
the case turns on (`responsabilità` · `anatocistici` · `principio` ·
`controversia`), keeping CS-TYPE-02 single-em editorial discipline intact.

**Default behavior**:
- Non-reduced-motion: cards ship with citation closed; click on summary
  toggles open inline. No modal. Modern browsers (Chrome 131+) animate the
  open/close transition via `::details-content`; older browsers fall back
  to instant native open. The interaction (click → inline expansion) is
  the differentiator, not the transition timing.
- Reduced-motion: `setupEvid3Reduced()` (added to live-motion.js init's
  reduced-motion branch) calls `d.open = true` on every `details[data-evid3]`
  on init, so the panel is rendered already-expanded as the static-with-
  collapse-link equivalent per F2-VAR-1.
